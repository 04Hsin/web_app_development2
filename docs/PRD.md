# 產品需求文件 (PRD)

## 1. 專案概述
- **專案名稱**：個人記帳簿系統
- **專案描述**：一個幫助使用者記錄日常收入與支出，並能顯示餘額統計的記帳應用程式。
- **背景與動機**：現代人常有不知錢花去哪裡的困擾。本系統旨在提供一個簡單易用的介面，讓使用者能快速記錄每筆收支，藉由統計數據掌握自身財務狀況，以達成更好的理財規劃。
- **目標用戶**：需要記錄日常收支、希望控管花費與掌握餘額的一般大眾。
- **核心價值主張**：簡單、快速、一目了然的記帳體驗，無須複雜設定即可了解資金流向。

## 2. 功能需求
以下為系統主要功能及其使用者故事：

1. **財務紀錄新增功能**
   - 作為 **記帳的使用者**，我希望 **能夠新增一筆收入或支出的紀錄（包含金額、日期、分類與備註）**，以便 **記錄我每一筆財務流向**。

2. **餘額與收支統計顯示**
   - 作為 **記帳的使用者**，我希望 **在首頁能清楚看到目前的總餘額、總收入與總支出**，以便 **快速掌握我現在的財務狀況**。

3. **收支分類管理**
   - 作為 **記帳的使用者**，我希望 **可以在記帳時選擇不同的收支分類（如：飲食、交通、薪水、娛樂等）**，以便 **知道我的錢都花在哪些種類上**。

4. **歷史紀錄查詢與管理**
   - 作為 **記帳的使用者**，我希望 **能夠查看過去的記帳歷史清單，並且能修改或刪除錯誤的紀錄**，以便 **維持帳目的正確性**。

5. **分類圖表與報表分析**
   - 作為 **記帳的使用者**，我希望 **可以查看各項分類支出的統計圖表**，以便 **檢視我的花費比例並進行未來的財務調整**。

## 3. 非功能需求
- **技術限制**：採用 Python Framework (Flask) 作為後端，前端頁面使用 Jinja2 搭配 HTML/CSS/JavaScript 呈現，資料庫使用 SQLite 儲存。
- **效能與安全考量**：
  - 使用者介面反應需迅速，提供流暢的使用體驗。
  - 對於表單輸入（金額、備註）必須進行基本的資料驗證與過濾（防範 XSS / SQL Injection 等）。
  - 若未來考慮支援多人使用，需具備帳號密碼。但在 MVP 階段可先實作單人使用的本地單機版。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：
  - 新增、修改、刪除收入與支出紀錄。
  - 顯示總收支與總餘額的統計面板。
  - 提供基礎的預設收支分類。
  - 顯示歷史明細清單。
- **Should Have (應該有)**：
  - 使用者能自訂收支分類。
  - 報表分析（如圓餅圖分析花費比例）。
- **Nice to Have (加分項目)**：
  - 支援多帳號註冊登入。
  - 匯出為 CSV 報表。
  - 提供預算設定功能。
